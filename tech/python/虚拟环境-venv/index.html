Python - 虚拟环境 Venv

发表于： 2023-06-01 更新于： 2023-06-02 分类于： Python
字数：698 阅读：≈ 2分钟 浏览：

原文请访问 小菠萝测试笔记(已授权) ，本文在此基础上进行部分修改，用于个人知识整理。

什么是虚拟环境

Python 虚拟环境主要是为不同 Python 项目创建一个隔离的环境，每个项目都可以拥有独立的依赖包环境，而项目间的依赖包互不影响。

  • 这是 Python 3.3 的新特性
  • 每个系统环境下可以创建多个虚拟环境
  • 默认情况下，虚拟环境和系统环境是完全隔离的
  • 每个虚拟环境都有自己的 Python 二进制文件及单独的包安装目录，与其他环境完全隔绝

为什么要使用虚拟环境

背景:

  • 假设需要开发多个 Python 项目
  • 每个项目所需的 Python 版本不一致，比如：Python 2.X、Python 3.X
  • 每个项目所需的依赖包不一致，比如 A 项目只要 Selenium 相关依赖包，B 项目只要 Appium 相关依赖包
  • 不同项目用同一个依赖包，但依赖包版本并不一致

如果不适用虚拟环境：

  • Python 第三方库默认都是装在系统固定目录的
  • 电脑上所有项目都会从这个目录去存储、查找第三方库（站点库）
# 以 Mac 为例
>>> import site
>>> site.getsitepackages()
['/usr/local/Cellar/python@3.9/3.9.5/Frameworks/Python.framework/Versions/3.9/lib/python3.9/site-packages']

带来的问题：

1、不同项目使用同一个依赖包，但版本不同时：

  • 当开发 A 项目时，要装一次依赖包 A 版本
  • 转开发 B 项目时，先要卸载依赖包 A 版本，再重装 B 版本

2、一个项目开发完成后，需要生成当前项目的所有依赖包 requirement.txt 时：

  • 会附加当前环境上包，但该包可能在项目里用不上
  • 需要人工检查，还得手工一个个删

通过 venv 操作虚拟环境

Python 3.5 后推荐使用 venv 来创建虚拟环境。

# 创建虚拟环境
$ python3 -m venv <环境名称>

# 激活虚拟环境
$ source <环境名称>/bin/activate
$ source venv/bin/activate
(venv)

# 确认已经切换到虚拟环境
$ which python
$ pip list
# 可以看到虚拟环境中的 Package 只有最基础的 pip、setuptools

# 在虚拟环境生成依赖包目录
$ pip freeze > requirements.txt

# 当完成工作并退出虚拟环境时，关闭虚拟环境
$ deactivate
