Python - Poetry 环境管理

发表于： 2023-05-30 更新于： 2023-06-02 分类于： Python
字数：1694 阅读：≈ 4分钟 浏览：

Poetry 是 Python 项目中创建虚拟环境、管理依赖包、打包发布的工具。

参考：

  • 官方文档
  • Github
  • Python 3.7+

优势：

  • 允许声明项目所依赖的库，并管理（安装/更新）
  • 提供了一个锁文件来确保可重复安装
  • 可以构建项目以进行分发
  • 强大的依赖处理能力

Poetry 以简单的项目格式取代：

  • setup.py
  • requirements.txt
  • setup.cfg
  • MANIFEST.in
  • Pipfile
  • pyproject.toml

安装

安装

# Linux、macOS、Windows (WSL)
$ curl -sSL https://install.python-poetry.org | python3 -

# Windows（PowerShell）
# 如果是通过 Microsoft Store 安装的 Python，将下面的 py 替换为 python
(Invoke-WebRequest -Uri https://install.python-poetry.org -UseBasicParsing).Content | py -

⚠️ 安装遇到问题

报错日志说是未在清华源中找到某个版本~ 取消掉镜像配置，重新安装。如何取消？（去 pip-包管理工具文档中找）

Mac 安装完成后，可执行命令默认在 ~/.local/bin/poetry

$ ~/.local/bin/poetry --version
Poetry (version 1.5.1)

# 由于 .local 并不在环境量下，所以还不能直接使用 poetry 命令
$ poetry --version
poetry: command not founds

# 复制命令到环境变量 或者 添加到环境变量
$ cp ~/.local/bin/poetry /usr/local/bin
$ poetry --version
Poetry (version 1.5.1)

更新

# 更新到最新版本
$ poetry self update

# 更新到指定版本
$ poetry self update 1.2.0

卸载

$ curl -sSL https://install.python-poetry.org | python3 - --uninstall
$ curl -sSL https://install.python-poetry.org | POETRY_UNINSTALL=1 python3 -

基本用法

创建新项目

创建方式一：默认项目

# 创建新项目
# 默认项目名和包名是一致的
$ poetry new default-project

# 查看项目结构
$ cd default-project
$ tree
.
├── README.md
├── default_project
│   └── __init__.py
├── pyproject.toml
└── tests
    └── __init__.py

创建方式二：指定代码路径及包名

# 指定项目名称为 project-demo
# 指定包名为 package-demo
# 指定包存放的目录为 src
$ poetry new project-demo --src --name package-demo
Created package package_demo in project-demo

$ tree
.
├── README.md
├── pyproject.toml
├── src  # <== 将包放在新增的代码目录内
│   └── package_demo
│       └── __init__.py
└── tests
    └── __init__.py

在已有的项目中初始化

$ cd exists_project/
$ poetry init     
# 填入自定义项，默认都是确定，回车就行
...
Generated file

pyproject.toml

以上项目创建或初始化完成后，都会在根目录生成 pyproject.toml 配置文件，在整个开发过程中，是逐步丰富此文件的。

这是项目的描述文件，管理项目及其依赖包，并描述了打包的配置信息。

[tool.poetry]
# 指定包名，默认与项目同名，如果创建时自定义了包名，那么写自定义的名称，而不是项目名
name = "package-demo"
version = "0.1.0"
description = "我是包的描述信息"
authors = ["lan <lanzy.nice@gmail.com>"]
readme = "README.md"
# 由于包名与项目名不同，所以要告诉项目包的位置在哪里；如果包名与项目名同名，则不用该配置
packages = [{include = "package_demo", from = "src"}]

[tool.poetry.dependencies]
# 这里指定该项目的所有依赖包，通过 add 安装包会自动填写在这里
python = "^3.10"


[build-system]
requires = ["poetry-core"]
build-backend = "poetry.core.masonry.api"

[tool.poetry.scripts]
# 通过包名可以直接调用该命令行执行
blog = "blog.cmdline:main"

当手工向配置文件添加内容后，可以通过命令来检查文件是否正常工作。

$ poetry check
All set!

激活虚拟环境

配置工作完成后，还需要给项目创建一个虚拟环境。

# 项目目录下
# 执行该命令后会创建并进入虚拟环境的终端
$ poetry shell
Creating virtualenv package-demo-4J6IgKuF-py3.10 in /Users/lan/Library/Caches/pypoetry/virtualenvs
Spawning shell within /Users/lan/Library/Caches/pypoetry/virtualenvs/package-demo-4J6IgKuF-py3.10

关于 env 的命令

# 查看当前环境信息
$ poetry env info

# 查看当前项目创建的环境列表
$ poetry env list
package-demo-4J6IgKuF-py3.10 (Activated)
(package-demo-py3.10)

# 切换环境
$ poetry env use /full/path/to/python
$ poetry env use 3  # 根据版本号

# 删除环境
$ poetry env remove /full/path/to/python

包管理

安装包

环境配置就绪，开始进行项目的开发工作，那么就需要安装一些项目的依赖包。

1、如果没有配置文件内没有配置任何依赖包，随用随装

# 安装包 并 自动添加到 配置文件中
$ poetry add click

# 指定版本
$ poetry add click@latest
$ poetry add click@^8.1.3
$ poetry add "click>=^8.1.3"

# Github 安装
$ poetry add git+https://github.com/sdispater/pendulum.git
$ poetry add git+https://github.com/sdispater/pendulum.git#master

# 本地安装
poetry add ./package-demo/

# -D 将包添加到开发依赖项中
$ poetry add -D requests

当使用了 -D 参数时，那么就会在 pyproject.toml 中生成一个新的依赖项，表示该软件包只会在开发期间使用，并且不应该被包含在生产环境中。

[tool.poetry.group.dev.dependencies]
pytest = "^7.3.1"

2、如果配置文件内已经指定好了项目需要的依赖包，安装已有项目

# 默认安装所有依赖包
$ poetry install

# 不安装开发依赖包
$ poetry install --no-dev

# 移除 poetry.lock 文件中不再存在的旧依赖包
$ poetry install --remove-untracked

Poetry 会简单地解析pyproject.toml文件中列出的所有依赖项并下载其文件的对应版本，默认为最新版本。

完成安装后，会在根目录写入文件 poetry.lock，锁定依赖包的特定版本，下次别人再安装就直接使用相同版本的依赖项。

更新包

当依赖项有更新时，需要使用 update 命令，等同于删除 .lock 然后重新执行 install 生成 .lock 文件。

# 获取所有依赖项的最新版本并更新 poetry.lock 文件
$ poetry update

# 仅更新指定依赖项
$ poetry update requests

# 不执行安装，仅更新 poetry.lock 文件
$ poetry update --lock

删除包

$ poetry remove click

# 从开发环境中删除包
$ poetry remove requests --dev

查看包

查看已安装的包，这里是根据 poetry.lock 列出。

# 查看项目中安装的所有包
$ poetry show 

# 查看单个包的信息
$ poetry show alembic

执行

执行 python 命令

$ poetry run python -V
Python 3.10.8
(package-demo-py3.10)

执行自定义的脚本

在 myproject.toml 中添加脚本的执行路径

[tool.poetry.scripts]
blog = "blog.cmdline:main"
  • 第一个 blog 为执行时调用的命令
  • 第二个 blog 为包名
  • cmdline 为定义命令行指令的 py 文件
  • main 为执行函数

构建

# 打包
$ poetry build

# 发布
$ poetry publish
