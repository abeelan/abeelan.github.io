参数化用例

单参数

import pytest

search_list = ["a", "b"]

@pytest.mark.parametrize("name", search_list)
def test_search(name):
   assert name in search_list

多参数

@pytest.mark.parametrize(
    "input_num, expect", 
    [("3+5", 8), ("2+3", 5), ("6+3", 9)],  # 多参数传递需要通过列表或元组嵌套的方式传递
    ids=["add1", "add2", "add3"]  # 用例标识，个数与前面的参数对应
)
def add(input_num, expect):
    assert eval(input_num) == expect

笛卡尔积

@pytest.mark.parametrize("letter", ['a', 'b', 'c'])
@pytest.mark.parametrize("number", [1, 2, 3])
def test_search(letter, number):
   print(f"{letter} - {number}")

标记测试用例

通过标记对测试用例进行分类

# test_demo.py

@pytest.mark.num
def test_num1():
    ...

@pytest.mark.num   
def test_num2():
    ...
    
@pytest.mark.str
def test_str1():
    ...

@pytest.mark.str
def test_str2():
    ...

# -vs 打印详细日志
$ pytest test_demo.py -vs -m "str"  # 仅执行 str 标记的测试用例

如果标签识别不到，需要在项目路径下新建 pytest.ini 文件，值有多个时，不要写在一行。

跳过（skip）及预期失败（xFail）

skip

始终跳过这条测试用例

import pytest

# 直接跳过 无任何输出信息
@pytest.mark.skip
def func():
    ...

# 直接跳过 说明原因 
@pytest.mark.skip(reason="代码未开发完")
def func():
    ...

skipif

遇到特定情况跳过该条测试用例

# 代码判断
def check():
    return True
    
def test_func():
    print("start")
    if check():
        pytest.skip("oh skip!")  # 如果用例被跳过，后续的 print 代码不会被执行
    print("end")  
# 装饰器判断
@pytest.mark.skipif(sys.platform=="darwin", reason="not run on mac")
def test_demo():
    ...

xfail

如果用例被标记为 xfail，那么：

  • 用例执行失败，被标记为 XFAIL，不会报错
  • 如果用例执行成功，会被标记为 XPASS
@pytest.mark.xfail(reason="bug not fixed")
def test_demo():
    ...
