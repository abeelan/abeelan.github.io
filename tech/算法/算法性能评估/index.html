算法 - 算法性能评估

发表于： 2023-08-14 更新于： 2023-08-15 分类于： 算法
字数：730 阅读：≈ 2分钟 浏览：

评估算法好坏的两个指标：

  • 时间复杂度
  • 空间复杂度

影响时间复杂度的因素：

  • 规模：测试数据的规模大小
  • 环境：运行环境速度的快慢

大O 表示法

def tmp(n):
  add = 0  # 1
  for i in range(n):  # n
    add += i  # n
  return 0

总共运行次数为 1+n+n 次，也就是 T(n)=(2n+1)*unit

通常，当 n 足够大时，低阶、常量、系数三部分并不左右增长趋势，所以都可以忽略。因此，用 O 表示渐近时间复杂度，表示代码执行时间随数据规模增长的变化趋势。上面代码时间复杂度记为：T(n)=O(n)

时间复杂度

1、只关注循环次数多的代码

回到上例，在整体代码中可以把循环次数（n）作为时间复杂度，即 O(n)

2、选大量级

def tmp(n):
  for i in range(999):  # n
    print(1)
  for i in range(n):  # n
    for j in range(n):  # n
      print(2)

存在多个循环次数的时候，选择最大量级的循环，如上例，两层循环。时间复杂度为 O(n*n)

3、嵌套循环要乘积

def tmp(n):
  for i in range(n):  # n
    a(i)  # n
    
def a(n):
  for i in range(n): 
    print(1)

类似双层循环，时间复杂度为 O(n*n)

空间复杂度

表示算法存储空间与数据规模之间的增长关系。

def tmp(n):
  a = [1] * n  # n 越大，a 所占用的空间越大
  for i in a:  # n
    print(i)

空间复杂度为 O(n)

数组数组与列表

数组是线性表数据结构。用连续的内存空间存储相同类型的数据。

线性表：线性表是数据排成一条线的结构，数据只有前后两个方向，包括：数组、链表、队列、栈 非线性表：数据之间不是简单的前后关系，包括：二叉树、堆、图等

连续的内存空间和相同的数据类型，使数组支持“随机访问”，通过下标就可以访问，访问速度很快。但是在数组中删除、插入数据时，需要做大量的迁移工作。

插入操作

  • 在数组末尾插入元素，不需要移动数据，最好情况时间复杂度为 O(1)
  • 在数组开头插入元素，所有数据需要后移一位，最坏情况时间复杂度为 O(n)
  • 在数组中间随机位置插入元素，平均时间复杂为 O(n)

删除操作

  • 删除末尾元素，最好情况时间复杂度为 O(1)
  • 删除首个元素，最坏情况时间复杂度为 O(n)
  • 删除中间元素，平均时间复杂为 O(n)
