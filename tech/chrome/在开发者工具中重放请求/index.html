Chrome - 在开发者工具中重放请求

发表于： 2023-04-14 更新于： 2023-08-16 分类于： Chrome
字数：506 阅读：≈ 2分钟 浏览：

接口根据请求参数不同，返回不同的响应值。

在 charles 中可以使用断点的方式，那么在谷歌浏览器的控制台中怎么实现呢？

  • 右键请求 - Copy - Copy as fetch
  • 切换到控制台 tab，粘贴
  • 修改参数后，按下回车
  • 返回 network tab 即可查看到重放请求

Copy as fetch

是指在 Chrome 浏览器的开发者工具中，可以将某个网络请求复制到剪贴板中，格式为 Fetch API 代码，以便在开发过程中进行调试和测试。

使用该功能，您可以轻松地将请求示例粘贴到 JavaScript 代码中，并通过更改参数来调整和测试请求。这方面的好处是使得开发人员不必手动编写 HTTP 请求代码，而是可以通过 Chrome 自动生成代码，从而减少了代码书写的时间和工作量。

Fetch API

Fetch API 是一种用于获取和发送 HTTP 请求的新型 Web API，它提供了一种现代化的方式来处理网络请求。Fetch API 提供了一组接口，可以让 JavaScript 代码轻松地处理HTTP请求和响应。

使用Fetch API，您可以向服务器发送 GET、POST 等各种类型的请求，并且可以控制请求头、请求体等参数。Fetch API 还支持 Promise，因此您可以轻松地使用异步方式处理请求和响应，而不必使用回调函数。

另外，值得注意的是，Fetch API 是基于浏览器内置的 XMLHttpRequest（XHR）对象实现的，但它提供了更为简洁、易用和灵活的 API 接口。
