Jmeter&Ms - 数据备份与恢复

发表于： 2023-06-01 更新于： 2023-08-15 分类于： Jmeter&Ms
字数：475 阅读：≈ 1分钟 浏览：

数据备份

MeterSphere 备份，主要是 MySQL 数据库的数据备份和 /opt/metersphere/data 路径下的目录备份。

备份内容：

  • mysqldump
  • 手动备份 /opt/metersphere/data/mysql

编写备份脚本 ms_backup.sh 

#!/bin/bash
                                                                                                              0,0-1         Al#!/bin/bas
backupDir=/data/backup/ms_db_bak

if [ ! -d "$backupDir" ]; then
  mkdir -p "$backupDir"
fi

currentTime=`date "+%Y-%m-%d-%H-%M-%S"`
ms_data=/opt/metersphere/data
ms_sql=$backupDir/ms_db_$currentTime.sql

echo "======= 数据库备份 ==============="
docker exec -i mysql mysqldump -uroot -pPassword123@mysql metersphere --max_allowed_packet=2G > $ms_sql
echo [Done] dumpSqlFilePath=$ms_sql
echo
echo "======= 备份数据打包 ============="
backupZipFileName=ms_db_$currentTime.zip
cd $backupDir
zip -r $backupZipFileName $ms_sql $data
# 打包完成后删除原来的 sql 备份
rm -rf $backupDir/ms_db_$currentTime.sql
echo [Done] backupZipFileName=$backupZipFileName
echo
echo "======= 仅保留最新 3 个备份 ======="
keepBackupNum=3
output=`ls -lt $backupDir/*.zip | awk '{print $9}'`
step=0
for backupFile in $output ;do
    step=$((step+1))
    echo step=$step
    echo $backupFile
    if [ $step -gt $keepBackupNum ];then
        echo Remove outdated backup $backupFile
        rm -rf  $backupFile
    fi
done

数据恢复

# 1. 进入备份 sql 目录，将 sql 复制到 mysql 容器的挂载目录
$ cp metersphere.sql /opt/metersphere/data/mysql

# 2. 进入 mysql 容器，登录数据库
$ docker exec -it mysql sh
$ mysql -uroot -pPassword123@mysql

# 3. 使用 metersphere 库，并将数据导入到库里
$ use metersphere;
$ source /var/lib/mysql/metersphere.sql

升级到 2.10.1 LTS 版本后遇到的问题

问题一：数据库迁移

容器报错信息

system-setting | Caused by: org.flywaydb.core.internal.command.DbMigrate$FlywayMigrateException: Schema metersphere contains a failed migration to version 137 !

参考地址： 升级报错

当前版本为 2.10.1 lts 版本，找到对应的 sql 脚本： V137__modify_novice_statistcs.sql 

# 将备份的sql数据复制至mysql容器
$ docker cp V137__modify_novice_statistcs.sql mysql:/

# 使用 metersphere 并执行 sql
$ docker exec -it mysql bash
$ mysql -uroot -pPassword123@mysql
$ use metersphere;

# 导入数据库文件
$ source /V137__modify_novice_statistcs.sql

# 更新版本表
$ update metersphere_version set success=1 where version=137;

# 退出容器；重启服务
$ msctl reload

问题二：prometheus 路径权限不够

容器报错信息

ts=2023-07-17T09:53:18.717Z caller=query_logger.go:91 level=error component=activeQueryTracker msg=“Error opening query log file” file=/prometheus/queries.active err=“open /prometheus/queries.active: permission denied” panic: Unable to create mmap-ed active query log

手动添加权限即可

$ chmod +777 /opt/metersphere/conf/prometheus
$ chmod +777 /opt/metersphere/data/prometheus
$ docker restart ms-prometheus
