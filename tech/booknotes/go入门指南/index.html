书中练习题

数组和切片

问题 7.3

给定 s := make([]byte, 5)，len(s) 和 cap(s) 分别是多少？s = s[2:4]，len(s) 和 cap(s) 又分别是多少？

  • 当执行 s = s[2:4] 后，len(s) 和 cap(s) 分别变为 2
  • 切片操作 s[2:4] 创建了一个新的切片 s，从原始切片 s 的索引 2 开始（包含）到索引 4 结束（不包含），因此新切片的长度为 2
  • 由于新切片是从原始切片的一部分创建的，所以其容量也是从原始切片的索引 2 开始（包含）到原始切片的末尾（索引 5）结束，即容量为 3

问题 7.4

假设 s1 := []byte{'p', 'o', 'e', 'm'} 且 s2 := s1[2:]，s2 的值是多少？如果我们执行 s2[1] = 't'，s1 和 s2 现在的值又分别是多少？

  • s2 的值为 []byte{'e', 'm'}，它包含了从原始切片 s1 的索引 2 开始（包含）到末尾的所有元素
  • 执行 s2[1] = 't'，则会修改 s2 切片的第二个元素（索引为 1），将其值改为字母 ‘t’
    • s1 = []byte{'p', 'o', 'e', 't'}
    • s2 = []byte{'e', 't'}
  • 切片是引用类型，修改 s2 的元素会影响到底层数组，因此也会反映在 s1 上。

如何理解 new、make、slice、map、channel 的关系

1.slice、map 以及 channel 都是 golang 内建的一种引用类型，三者在内存中存在多个组成部分，需要对内存组成部分初始化后才能使用，而 make 就是对三者进行初始化的一种操作方式

2. new 获取的是存储指定变量内存地址的一个变量，对于变量内部结构并不会执行相应的初始化操作， 所以 slice、map、channel 需要 make 进行初始化并获取对应的内存地址，而非 new 简单的获取内存地址。

练习 7.5

给定切片 sl，将一个 []byte 数组追加到 sl 后面。写一个函数 Append(slice, data []byte) []byte，该函数在 sl 不能存储更多数据的时候自动扩容。

package main

import "fmt"

func main() {
	slice := []byte{1, 2, 3}
	data := []byte{4, 5, 6}
	slice = Append(slice, data)
	fmt.Println(slice)
}

func Append(slice, data []byte) []byte {
	// 计算新切片的长度
	newLen := len(slice) + len(data)

	// 如果容量不够 则进行倍增扩容
	if newLen < cap(slice) {
		newCap := cap(slice) * 2
		if newCap < newLen {
			newCap = newLen
		}
		// 创建新切片 复制原始切片进来
		newSlice := make([]byte, newLen, newCap)
		copy(newSlice, slice)
		// 追加新数据至末尾
		newSlice = append(newSlice, data...)
		return newSlice
	}

	// 如果容量足够 则直接增加
	slice = append(slice, data...)
	return slice
}

问题 7.5

假设我们有如下数组：items := [...]int{10, 20, 30, 40, 50}

a) 如果我们写了如下的 for 循环，那么执行完 for 循环后的 items 的值是多少？如果你不确定的话可以测试一下:)

for _, item := range items {
	item *= 2
}

// item 是取出的切片中的元素的副本
// 因此，在循环体内对 item 进行修改不会影响原始的数组 items
// items 的值仍然是 [10, 20, 30, 40, 50]

b) 如果 a) 无法正常工作，写一个 for 循环让值可以变成自身的两倍。

for i := range items {
	items[i] *= 2
}
// 使用索引 i 来访问数组元素，并将其乘以 2

问题 7.6

通过使用省略号操作符 ... 来实现累加方法。

func sum(nums ...int) int {
	total := 0
	for _, num := range nums {
		total += num
	}
	return total
}

func main() {
    result := Sum(1, 2, 3)       // 结果为 6
    result = Sum(10, 20, 30, 40) // 结果为 100
    result = Sum(5)              // 结果为 5
    result = Sum()               // 结果为 0，没有参数时返回初始值
}

练习 7.16

编写一个程序，使用冒泡排序的方法排序一个包含整数的切片。

package main

import "fmt"

func main() {
	sla := []int{2, 6, 4, -10, 8, 89, 12, 68, -45, 37}
	fmt.Println("before sort: ", sla)
	BubbleSort(sla) // 引用传递
	fmt.Println("after sort:  ", sla)
}

func BubbleSort(slice []int) {
	// 确定总共需要排序的个数
	for pass := 1; pass < len(slice); pass++ {
		// 每轮将最大数换到最后
		for i := 0; i < len(slice)-pass; i++ {
			if slice[i] > slice[i+1] {
				slice[i], slice[i+1] = slice[i+1], slice[i]
			}
		}
	}
}

包

问题 9.1

a）一个包能分成多个源文件么？

一个包可以分成多个源文件。在 Go 中，一个包可以由多个以 .go 扩展名结尾的源文件组成。这种方式可以帮助将相关的代码组织在一起，提高代码的可维护性和可读性。这些源文件可以包含包级别的声明，如常量、变量、函数和结构体等，并共享同一个包名。

b）一个源文件是否能包含多个包？

一个源文件只能属于一个包。每个源文件的开头需要通过 package 声明来指定它所属的包。一个包通常对应一个目录，而每个目录下只能包含一个包。这是 Go 语言的约定，用于管理代码结构和避免命名冲突。因此，一个源文件只能包含一个包的定义。如果需要创建多个包，就需要分别放置在不同的源文件中。
