Docker - 构建镜像命令

容器更新为镜像

如果容器中安装的新包需要更新为镜像，请执行以下步骤：

  1. 确保容器中安装的包已保存到容器的镜像层中。可以通过执行以下命令来检查容器的更改：
# 将显示容器中进行的任何更改，例如添加，删除或修改的文件
$ docker diff <container-id>
  1. 在容器运行时，使用 docker commit 命令将容器转换为新的镜像，此命令将使用容器的当前状态创建一个新的镜像层，并将其添加到现有的镜像中。
$ docker commit my-container my-image
  1. 使用 docker images 命令来检查新的镜像是否已添加到本地 Docker 镜像列表中：
$ docker images
# 应该能够看到 my-image 已添加到列表中。
  1. 将新的镜像上传到 Docker 仓库，以便其他人可以使用：
$ docker push my-image

这将将 my-image 推送到默认Docker仓库中（即 Docker Hub）。可以在 docker push 命令中指定仓库名称，以将新镜像推送到仓库中。
