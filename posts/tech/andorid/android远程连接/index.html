Android - 设备远程连接方式

背景#

在一些移动云测手机机房里 Android 设备是分批挂载在专门的宿主机集群上的，而执行自动化脚本的机器是在另外一个执行机集群中，因此就会产生这样的需求：执行机上的自动化脚本需要通过网络远程连接的方式来控制宿主机上的 Android 设备。

今天分享两个实现远程连接 Android 设备的方案。

实现#

ADB Host 模式#

手机连接电脑，执行如下命令

$ adb kill-server && adb -a -P 5037 nodaemon server

然后在执行脚本的服务器上通过下面的命令远程连接设备：

# {device_hub_ip} 是指挂载 Android 设备的宿主机 IP
$ adb -H {device_hub_ip} -P {port} {其他 adb 命令}
# 如果是原生命令，则通过下面的方式连接使用
$ adb -H 10.x.x.x -P 5037 devices

ADB WIFI 连接#

1、将手机通过USB连接上手机，打开手机的USB调试模式，通过下述命令确认设备连接成功：

$ adb devices

2、打开adb监听端口，直接在电脑上运行

adb tcpip 5555

这里的端口号可以指定其他值，该命令将会重启手机上的adbd，开启网络调试功能。

3、远程连接，断开手机和PC的有线连接，直接使用 adb 命令远程连接：

$ adb connect ip:port

注意这里的IP是手机的IP，如果想要通过宿主机的IP连接，那么需要再做一次端口转发：

adb kill-server && adb -a -P 5037 nodaemon server
adb -H 10.x.x.x tcpip 5555
adb -H 10.x.x.x forward tcp:50001 tcp:5555
adb connect 10.x.x.x:50001

这时候再通过devices命令查看，就可以看到连接好的设备：

adb devices

4、断开远程连接

adb disconnect ip:port
